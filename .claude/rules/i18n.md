# i18n rules for tomscholtes.com

Three target locales added to English: **DE**, **FR**, **RU**. Layout, theme, spacing, typography, and hover animations MUST be visually identical across all four locales.

## Locale model

- English is the canonical source. Source of truth for keys.
- Locales: `en` (canonical), `de`, `fr`, `ru`.
- Dictionary location: `src/i18n/<locale>.json`. Flat keys, dot-namespaced (e.g., `"nav.about"`, `"home.hero.title"`).

## Architectural rules

- Use **Astro 5's built-in i18n routing** (`i18n: { defaultLocale, locales, routing: { prefixDefaultLocale: false } }` in `astro.config.mjs`).
- Strings flow through a small `t(key)` helper imported from `src/i18n/index.ts`. Components consume strings via `t()`, never hardcode.
- Add a `<LangSwitcher />` Astro component in the header. Three plain links, no JS island. Active locale indicator via CSS class only.
- URL strategy: `/` for English, `/de/`, `/fr/`, `/ru/` for others. `prefixDefaultLocale: false`.

## What translators may edit

- `src/i18n/de.json`, `src/i18n/fr.json`, `src/i18n/ru.json` — values only. Keys are frozen by the architect / frontend pass.

## What translators may NOT edit

- Components, layouts, pages, styles, build scripts, astro.config.mjs, en.json.

## String hygiene

- Placeholders `{name}`, `{count}` survive untouched.
- Markdown / inline HTML survives untouched.
- Em-dashes (U+2014) FORBIDDEN in any locale. Use periods, semicolons, or rewrite.
- Personal / workplace names from the leakage list FORBIDDEN. See `~/Projects/Personal_Website/V5/NOTES_GUIDE.md` for the full list.
- Brand names stay literal across locales (e.g., "Tom Scholtes"; do not transliterate to Cyrillic for RU unless the source has it transliterated).
- Technical terms in English unless an established native term exists in current professional use (e.g., "Software Engineer" stays English in DE; "Anwalt" not "Lawyer" for legal).

## Locale-specific register

- **DE:** Sie-form throughout. Avoid Anglicisms unless they are the established term.
- **FR:** vouvoiement throughout. Luxembourgish / Belgian French norms where they diverge from France.
- **RU:** modern professional register, «вы» form, не калька с английского.

## Layout invariance check

After translations land, screenshots of `/`, `/de/`, `/fr/`, `/ru/` must show identical layout: same hero height, same column width, same font sizes, same spacing, same hover behaviour. Any divergence = bug in component, not in translation.

## Build gates

- `scripts/check-notes.mjs` em-dash + leakage grep must pass across all four locales.
- An additional `scripts/check-i18n.mjs` (to be created by frontend) should verify: every key in `en.json` exists in `de.json`, `fr.json`, `ru.json` (no missing translations); no extra keys.
